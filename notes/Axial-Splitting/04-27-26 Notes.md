## How the commit was made

A `cleanup.sh` script lives at the repo root. Runs idempotently and handles GitHub size limits by stripping notebook outputs as a fallback before retrying push. Why a script and not direct sandbox execution: the FUSE-mounted `.git/` from the sandbox blocks unlink of `.git/index.lock`, so any C-git index operation hangs the lock; auth to push is also keychain-only. Running `bash ~/Seismology/axial-splitting-ml/cleanup.sh` from a Mac terminal does the whole pass.
