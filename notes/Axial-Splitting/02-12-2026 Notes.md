## Steps

### 1. Read MFAST, SWSPy original papers and documentation. How are start and end windows determined?

**MFAST:** In their example study, at a volcano, they suggest 2–5 start times and ≥10 end times. MFAST builds on Teanby by grading clusters on multiple characteristics, rather than just minimal variance, and using the frequency of the seismogram to determine the lengths of analysis windows.

They use 5 start times, `Nwbeg`. They used time-step size between window start times `dTbeg = 0.2` at start, decreased to the smaller of `0.1 s` or `((S-P)/2) / Nwbeg`. They use time-step size between window end times `dTend` to start at `0.08 s`, increasing or decreasing to make `Nwend` between 15 and 25. The number of end times tested, `Nwend = int((w4 - w3) / dTend)`. The desired last window end time, `w4 = Stime + Tmid × 2.5 + 0.15`. So it's 2.5 cycles of the dominant frequency plus a little 0.15 s buffer at the end. `Tend0 = w3` is the earliest window end time, defined as `Stime + Tmid/1.2 + 0.15`. So it's ⅚ of the first cycle plus a little buffer. This makes sense — the number of end times is proportional to the total range of end times divided by the step size.

The number of start times is just set. Why? They pick 5 start times that will essentially go from just before the S-pick time to just after, scaling by the S-P time to avoid P-wave contamination.

The first time to start a window, `Tbeg0`, is defined as 0.3 seconds before the S-pick, but decreased to the smaller of `0.1` or `((S-P)/2) / (Nbeg + 1)`. The last time to start a window is `Tbeg1 = S_time − (Tbeg0 + (Nbeg × dTbeg))`. So the window start times go from 0.1 seconds before the S-pick or a fraction of the S-P time before the S-pick, to that value plus the number of beginning start times times the number of step sizes for the start windows. Again, `dTbeg` is calculated as 0.1 or a fraction of the S-P time.

**SWSPy:** The user specifies the S-wave arrival time uncertainty, called `win_S_pick_tolerance`, the earliest possible start time of any window, called `overall_win_start_pre_fast_S_pick`, and the earliest possible end time of any window, called `overall_win_start_post_fast_S_pick`. These are all defined relative to the S-pick time, so the uncertainty is quite important. They note that there are methods of automatically calculating the windowing parameters using spectral estimates, but that is not currently implemented in SWSPy, and they leave it to future contributors who deem it important enough to implement.

It should be noted that `win_S_pick_tolerance` is one value, but it also designates the time before and after the S-pick to not allow windows to start within, in seconds. So this is a ± buffer around the S-pick that shifts the window start times. `max_t_shift_s` is the maximum time to shift the data by, in seconds. `n_win` is the number of start and end times to pick, which is currently constant window step sizes within the specified range.

They compare their results to similar packages, including MFAST, and find identical splitting results within uncertainty.

What's not immediately obvious is whether the window lengths actually vary. What is the time-step size? Is it the same time-step size between the possible start times and possible end times?

Based on Fig. 4 in the main shear-wave splitting paper, it looks like you have S-pick minus S-pick tolerance, and you have the first pre-arrival window time, `overall_win_start_pre_fast_S_pick`, and you define a number of windows. Then all the window start times are each window in the range `(S-pick tolerance − overall_win_start_pre_fast_S_pick) / n_win`. Which, in their example, is `(0.2 s before S-pick − 0.1 s before S-pick) / 10`. So that length is 0.1 seconds, and there is a window start time every 0.01 seconds. Then the end windows have the same number of windows and same effective step sizes as the starting windows, with the first window end time at `overall_win_start_post_fast_S_pick`. In their example, that's 0.3 seconds after the S-pick. The end times then end 0.1 seconds afterwards, with 10 possible end times in that period. So now you have 10 window start times and 10 window end times, and each combination of windows is a window for analysis. The longest window, in their example, goes from 0.2 s before the S-pick to 0.4 s after the S-pick. The shortest window goes from 0.1 s before the S-pick to 0.3 s after the S-pick.
